# 余薪 Yuxin

[![CI](https://github.com/wnma3mz/yuxin/actions/workflows/ci.yml/badge.svg)](https://github.com/wnma3mz/yuxin/actions/workflows/ci.yml)

一个每秒更新的终端仪表盘：看看今天赚了多少、多久下班、离退休还有多远，以及当前余额每天能花多少。

## 直接运行

从 [GitHub Releases](https://github.com/wnma3mz/yuxin/releases) 下载对应 ZIP，解压后运行。Yuxin 是原生单文件程序，目标电脑不需要安装 Go、Python 或其他运行时。

| 系统 | 文件 |
| --- | --- |
| Apple Silicon Mac | `yuxin-v0.1.0-macos-arm64.zip` |
| Intel Mac | `yuxin-v0.1.0-macos-x86_64.zip` |
| 64 位 Windows | `yuxin-v0.1.0-windows-x86_64.zip` |
| 64 位 Linux | `yuxin-v0.1.0-linux-x86_64.zip` |

macOS 或 Linux 首次运行：

```bash
chmod +x yuxin
./yuxin
```

Windows 直接运行 `yuxin.exe`。如果 macOS 拦截未签名程序，请先核对 Release 中的 SHA-256，再按系统提示授权。

首次启动不会提问。程序会创建 `~/.config/yuxin/config.toml`，使用默认配置直接进入仪表盘。

## 可以看到什么

- 今日收入按秒增长，同时展示工时、下班倒计时和工作进度。
- 根据出生日期、性别和渐进式延迟退休规则估算退休年月。
- 汇总银行卡、定期存款等现金账户，计算实时余额和“现在退休每天可花”。
- 使用随程序发布的年度节假日数据展示假期进度，刷新时不联网。
- 自动适配宽屏、窄屏和低高度终端；管道输出自动切换为单次快照。

默认值是月薪 8,000 元、每月 22 个工作日、余额 10 万元、1995-01-01 出生、男性、周一至周五 09:00–18:00、午休 12:00–13:00、每秒刷新。

## 修改配置

在仪表盘按 `e`，或者运行：

```bash
yuxin config
```

配置按模块修改，不会要求重新填写全部内容：

```text
1 薪资      月薪 ¥8,000.00
2 工作时间  09:00–18:00，周 1,2,3,4,5
3 刷新      1s
4 退休      1995-01-01，男
5 资产      1 个账户，合计 ¥100,000.00
```

金额支持 `20w`、`20万`、`200k` 和 `200,000`。退休资料可以手动填写，也可以输入身份证自动解析出生日期和性别；程序只保存解析结果，不保存身份证原号码。女性还需要选择原法定退休年龄 50 岁或 55 岁的人员类型。

## 常用命令

```text
yuxin                 启动每秒刷新的仪表盘
yuxin once            输出一次无 ANSI 快照
yuxin config          分模块修改配置
yuxin doctor          检查配置和本地数据
yuxin --interval 2    本次改为每两秒刷新
yuxin --config FILE   使用指定 TOML 文件
yuxin --version       显示版本号
```

仪表盘快捷键：`e` 配置、`r` 立即刷新、`d` 计算口径、`?` 帮助、`q` 退出。

## 数据与计算口径

所有配置都保存在本机。Yuxin 不连接银行、企业系统或其他账户，也不会在每秒刷新时调用外部 API。默认配置和年度节假日 JSON 已嵌入可执行文件，Release ZIP 里也会附带一份便于查看。

```text
实时余额 = 所有账户余额 + 今日已赚工资
可支配余额 = 实时余额 - 应急保留金
现在退休每天可花 = 可支配余额 ÷ 距退休日历天数
退休进度 = 已出生天数 ÷ 出生日至预计退休日的总天数
```

退休日期按 2025 年起实施的渐进式延迟法定退休年龄规则估算。结果不包含个税、奖金、利息、通胀、养老金和未来政策变化，不构成财务建议。

## 从源码构建

需要 Go 1.22 或更高版本，项目只使用标准库：

```bash
go test ./...
go vet ./...
go build -trimpath -ldflags="-s -w -buildid=" -o yuxin .
./yuxin
```

终端设计说明见 [docs/terminal-ui.md](docs/terminal-ui.md)，版本变化见 [CHANGELOG.md](CHANGELOG.md)。

## 发布

推送版本标签后，GitHub Actions 会测试并构建 macOS、Windows 和 Linux 单文件程序，附带默认配置、节假日数据和 SHA-256：

```bash
git tag v0.1.0
git push origin v0.1.0
```

标签版本必须与根目录 `VERSION` 中的版本一致。
