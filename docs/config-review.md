# Yuxin 配置逻辑

本文记录 v0.6.0 的完整配置入口、交互顺序、保存规则、兼容行为和数据清理方式。产品只考虑终端用户，不提供 Web 或桌面配置入口。

## 1. 配置入口

### 1.1 首次启动

- 普通启动时，如果默认配置不存在，程序自动创建配置文件，不向用户提问。
- 创建完成后直接进入仪表盘。
- 默认退休倒计时、存款和隐私模式关闭。

### 1.2 仪表盘

- 在交互式仪表盘按 `e` 进入配置。
- 配置结束后返回仪表盘，并使用刚保存的配置重新计算。
- `--interval` 只覆盖本次运行；进入配置不会把临时刷新间隔写入文件。

### 1.3 独立命令

- `yuxin config`：进入同一套配置菜单，完成后返回命令行。
- `yuxin config export FILE`：导出当前配置。
- `yuxin config import FILE`：校验并导入配置。
- `yuxin config clear`：确认后删除当前配置。
- `yuxin doctor`：检查配置路径、终端模式和节假日数据。
- `yuxin doctor --strict`：缺少本年度节假日数据时返回非零状态，便于自动化检查。

### 1.4 配置路径

配置路径按以下优先级确定：

1. `--config FILE`
2. 环境变量 `YUXIN_CONFIG`
3. 可执行文件同目录的 `yuxin.toml`（ZIP 便携模式）
4. `$XDG_CONFIG_HOME/yuxin/config.toml`
5. `~/.config/yuxin/config.toml`

便携配置按可执行文件的位置查找，不依赖启动时的工作目录；在仪表盘修改配置时也会保存回该文件。这使直接下载的 ZIP 可随整个目录移动，但薪资、存款和退休信息也会保存在该目录中；分享或删除前需要先检查。

显式指定的配置文件不存在时，普通运行会报错；`yuxin config` 会创建一份默认配置供用户修改。

## 2. 当前主菜单

```text
当前配置
  1 今日入账   月薪 ¥8,000.00
  2 工作时间   09:00–18:00，周 1,2,3,4,5
  3 退休倒计时 关闭
  4 存款       关闭
  5 更多设置   刷新 1s
选择要修改的部分 [0]:
```

- 输入 `1` 到 `5` 修改对应部分。
- 输入 `0` 或直接回车完成。
- 修改完成后校验整份配置，成功才原子保存。
- 进入某项但最终值没有变化时，显示“未修改”，不重写配置文件。
- 输入格式错误时留在当前问题继续输入。
- 模块最终组合无效时不保存，并返回主菜单。

## 3. 各配置项

### 3.1 今日入账

当前按顺序询问：

1. 薪资模式：月薪、日薪、时薪或年薪。
2. 对应金额。
3. 月薪模式下的每月工作天数。

规则：

- 金额必须大于 `0` 且不超过 `1,000,000,000,000`。
- 每月工作天数必须在 `0` 到 `31` 之间（不含 `0`）。
- 支持 `20w`、`20万`、`200k` 和 `200,000`。
- 切换薪资模式时，默认金额会按当前每月工作天数和净工时等价换算并取整，用户仍可覆盖。
- 日薪、时薪和年薪配置仍保留 `monthly_workdays`，用于薪资模式换算。

### 3.2 工作时间

当前按顺序询问：

1. 工作日，例如 `1,2,3,4,5`。
2. 上班时间。
3. 下班时间。
4. 午休时长（分钟）。

规则：

- `1` 表示周一，`7` 表示周日，至少选择一天。
- 时间必须使用 `HH:MM`。
- 下班时间必须晚于上班时间。
- 午休时长输入 `0` 表示不扣除，正整数表示扣除的分钟数，且必须小于当日工作时长。
- 新编辑默认从 12:00 开始推导午休区间；12:00 不在工作时段时，自动把午休居中放入工作时段。
- 配置文件仍保留 `lunch_enabled`、`lunch_start` 和 `lunch_end` 以兼容旧文件，新交互不再询问开始与结束。

### 3.3 退休倒计时

退休倒计时只保留自动估算：

```text
退休倒计时：0 关闭 / 1 开启自动估算 [1]:
年龄或出生年月（如 30 / 1995-06） [30]:
性别：1 男 / 2 女 [1]:
```

- 在模式选择输入 `0` 关闭退休倒计时。
- 年龄接受 `1` 到 `100` 的整数；程序以当天的月日反推估算生日。
- 出生日期接受 `YYYY-MM`、`YYYY-MM-DD`、斜杠、点号和中文年月日写法；只填年月时日期取该月 1 日。
- 自动模式性别默认男；女性新配置按原 55 岁口径估算，旧配置中的原 50 岁口径继续兼容。
- 退休距离分三行显示剩余总年数、总月数和总天数；年和月向下取整，天数保持精确整数，不再提供 `u` 单位切换。
- 退休进度从 18 岁起算，不询问参加工作年份。

保存新退休配置时：

- `profile_enabled = true`；`[profile]` 只写入 `birth_date` 和 `sex`。
- `retirement_years = 0`，`retirement_mode = "full"`。
- 兼容字段 `progress_birth_date` 只写当天占位，不重复保存用户出生日期；关闭退休时删除 `[profile]` 并清理用户值。
- 新女性配置不写 `female_track`；程序把缺省值解释为原 55 岁口径。

### 3.4 存款

询问存款金额，开启后可选择一种存款目标：

```text
存款金额（0 关闭） [0]:
存款目标：0 关闭 / 1 躺平月支出 / 2 心愿物品 [1]:
```

- 接受大于或等于 `0` 的金额。
- 存款输入 `0` 关闭存款显示，并同时清除账户数组和存款目标。
- 输入正数时只保存一个名为“存款”、类型为 `deposit` 的条目。
- 不再询问账户名称、账户类型、应急保留金或多账户明细。
- 编辑存款时会把旧版多账户配置合并为用户本次输入的单一存款金额，并把应急保留金归零。
- 仪表盘的“实时存款余额”为手动存款，加上从起算日到今日之前按当前薪资、工作时间和可用节假日数据回算的完整工作日工资；当天部分按刷新间隔跳动。
- 程序不保存每天的历史账本。修改存款金额会把起算日更新为当天；金额未变时不重置。修改薪资或工作时间会让已有区间按新配置重新估算。
- 退休开启时同时显示用当前存款撑到退休每天和每月可花金额；年度换算不在主界面重复展示。退休关闭时只显示实时余额。
- 躺平目标只填“目标每月可花”，默认为 `3,000`；日、年口径由程序换算。
- 心愿目标只填物品名称和一次性目标金额，不询问目标日期、分期或收益率。
- 两种目标互斥，切换目标类型会清除另一种目标值，避免主界面同时出现两套口径。
- 日预算为实时余额除以距退休日历天数；月预算按 30.436875 天换算。
- 躺平目标依赖预计退休日期，只有存款和退休倒计时都开启时才显示；心愿目标只依赖存款。
- 目标总额 = 目标月支出 × 距预计退休日期的日历天数 ÷ 30.436875，表示“现在停止工作，用存款撑到退休”。
- 仪表盘把月目标自动换算为每天、每月、每年可花，并显示存款进度条和还差金额；达成后显示“已达成”。不引入寿命、收益率、通胀或养老金假设。
- 心愿进度 = 实时可用存款 ÷ 目标金额，并显示“距离拿下还差”；达成后显示“已达成”。
- 隐藏金额时，目标名称、金额、还差金额和进度比例一并隐藏。

### 3.5 更多设置

先选择修改刷新间隔或自定义口号：

- 刷新间隔单位为秒，允许范围为 `0.1` 到 `3600` 秒。
- `--interval SECONDS` 可以只覆盖本次运行，不修改文件。
- 口号默认为“摸鱼有数，下班有期。”，最长 40 个字，在 70 列以上终端的状态栏中居中显示。

## 4. 仪表盘隐私快捷键

隐私模式不属于配置菜单。它与“配置”和“退出”同级：

```text
[e] 配置  [p] 隐私  [r] 刷新  [v] 视图  [q] 退出
```

每按一次 `p` 立即切换到下一档并保存。宽屏底栏会持续显示“隐私·金额”或“隐私·全部”，默认关闭时显示“隐私”；窄屏自动退回短标签：

```text
关闭 → 隐藏金额 → 全部隐藏 → 关闭
```

- 档位 1 同时设置 `hide_amounts = false` 和 `hide_retirement_date = false`。
- 档位 2 设置 `hide_amounts = true`、`hide_retirement_date = false`，隐藏今日入账、今日预计、实时存款余额、退休可花预算以及心愿目标的名称和进度。
- 档位 3 两个字段都设为 `true`，在档位 2 基础上隐藏全部退休数据，避免从倒计时反推退休日期。
- 隐私只影响显示，不删除薪资、存款或退休配置。`yuxin share` 默认合成数据不依赖此档位；显式使用 `--real` 时继续遵守当前隐私档位。

## 5. 配置文件操作

### 5.1 导出

- 导出前提示文件可能包含敏感信息。
- 目标路径转换为绝对路径。
- 目标是目录时拒绝导出。
- 目标文件已存在时直接覆盖。
- 导出会使用当前程序重新序列化配置，不保证与源文件逐字一致。

### 5.2 导入

- 源文件必须存在且是普通文件。
- 源文件和当前配置不能是同一文件。
- 先完整解析并校验，成功后才原子替换当前配置。
- 导入会覆盖当前配置，不自动备份。
- 旧版退休和多账户配置仍可导入，兼容规则见第 8 节。

### 5.3 清除

- 显示即将删除的绝对路径。
- 必须准确输入大写 `DELETE`。
- 其他输入、空输入或输入结束都视为取消。
- 只删除配置文件，不删除导出的备份。
- 下次普通启动会重新创建默认配置。

## 6. 卸载

### 6.1 保留配置

```text
yuxin uninstall
```

- 必须在交互式终端运行。
- 显示当前可执行文件的绝对路径。
- 必须准确输入大写 `UNINSTALL`。
- 只删除当前可执行文件，不推断并删除整个目录。
- 本地配置继续保留。

### 6.2 同时清除配置

```text
yuxin uninstall --purge
```

- 显示程序路径和当前解析到的配置路径。
- 必须准确输入大写 `PURGE`。
- 先验证程序与配置路径安全，再执行卸载。
- 只清除当前配置文件，不删除用户主动导出的备份。
- Windows 在程序退出后由临时 PowerShell 脚本删除自身；默认脚本安装目录还会从用户 `PATH` 中移除。

## 7. 保存与安全规则

- Unix 类系统中配置目录权限为 `0700`，配置文件权限为 `0600`。
- 保存时在同一目录创建私有临时文件，写入并同步成功后原子替换目标。
- 配置文件最大 1 MiB。
- 未知字段、重复字段、非法类型和非法组合全部报错，不静默忽略。
- 资产条目仍保留最多 100 项和名称安全限制，只用于兼容旧配置；新交互只会写一个“存款”条目。
- 配置写入前始终执行完整校验。

## 8. 旧配置兼容

当前继续读取旧版配置中的兼容字段，避免升级后丢失数据。

### 8.1 旧版轻量退休

- 以 `retirement_years` 保存的预计剩余年数仍可读取和显示。
- 一旦用户重新编辑退休，新值转为出生日期与性别自动估算。
- 旧 `[profile]` 中的 `female_track = "50"` 或 `"55"` 仍可读取；新配置界面不再询问该字段。

### 8.2 旧版多账户资产

- 多个 `[[assets]]` 仍可读取并求和。
- 未编辑时保持兼容显示总额。
- 一旦用户编辑“存款”，旧账户会折叠为一个存款条目。

### 8.3 旧版分项隐私

- 只隐藏金额或只隐藏退休年月的文件仍可读取。
- 只隐藏金额对应档位 2；只隐藏退休年月的旧值在菜单中默认对应档位 3，再次保存时会同时隐藏金额。

## 9. 配置文件结构

文件版本仍为 `version = 1`；已开启存款且缺少起算日的旧配置，首次运行时会以当天补写。主要字段：

- 顶层：刷新间隔、自定义口号、躺平目标月支出、心愿名称与金额、存款累计起算日、旧轻量退休字段、兼容开关、应急保留金。
- `[salary]`：薪资模式、金额、每月工作天数。
- `[schedule]`：工作日、上下班时间、午休。
- `[privacy]`：两个底层隐藏字段，映射为关闭、隐藏金额、全部隐藏三档。
- `[profile]`：保存出生日期与性别，并兼容旧女性口径字段。
- `[[assets]]`：新配置只写一个存款条目，读取时兼容旧版多账户。

保留旧字段的目的是兼容现有用户文件，不表示这些复杂选项仍会出现在配置菜单中。
