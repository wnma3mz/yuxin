# 余薪演示与宣传视频计划

## 已确定方向

- 由 Go 直接生成确定性的终端帧记录，使用 agg 渲染 README GIF。
- 首帧直接展示完整主页，不使用空白终端或品牌片头作为封面。
- README GIF 与横版宣传片只使用 `DemoDashboard` 固定合成数据，不读取用户配置，不访问网络。
- 竖屏宣传片使用 Chrome/Chromium 截取当前 Web 前端，只读取公开聚合接口，不提交数据。
- README GIF、横版宣传片和竖屏宣传片是三个独立交付物；横版与 GIF 复用 TUI 演示，竖版固定展示浏览器前端。
- Chrome Canvas 与 macOS 原生 Go 渲染器归档保留，只作为视觉和技术参考。

## 两类交付物

| 交付物 | 用途 | 建议规格 | 内容重点 |
| --- | --- | --- | --- |
| README GIF | GitHub 首页快速了解产品 | 约 15 秒、循环、无音频、控制在 1 MB 内 | 主页、隐私切换、回到完整仪表盘 |
| 横版宣传视频 | Release 和项目介绍 | 1920×1080、约 20 秒、H.264 MP4、控制在 12 MB 内 | TUI 仪表盘、匿名看板、品牌收尾 |
| 中文竖屏宣传视频 | 抖音、TikTok 等竖屏平台 | 1080×1920、约 22 秒、H.264 MP4、控制在 12 MB 内 | 真实浏览器前端、公开聚合看板、匿名回声、品牌与网址 |

README 只嵌入 GIF。MP4 作为评审和宣传素材，不直接嵌入 README。

## README GIF 分镜

1. **主页（0–9 秒）**
   - 首帧即完整仪表盘。
   - 画面同时包含今日入账、下班倒计时和节假日进度，先让用户理解核心用途。
   - 演示时间、今日入账和实时存款每秒重新计算，呈现真实刷新效果。
2. **隐私切换（9–12 秒）**
   - 切换到“隐藏金额和存款”，敏感数字变为占位符。
   - 底部状态同步显示隐私档位，证明截图前可以快速保护数据。
3. **完整仪表盘（12–15 秒）**
   - 从隐私模式回到完整合成演示，收束主要能力。
   - 保留“离线本地运行、数据绝不上云”。
4. **循环回主页**
   - 结尾不追加长说明，直接回到主页，避免循环时突兀。

## 完整宣传视频分镜

### 横版：TUI

1. **情绪引入（0–3.4 秒）**
   - 以“这班不能白上，至少得知道赚了多少”建立打工人视角。
2. **完整仪表盘（3.4–12.4 秒）**
   - 在同一页面展示今日入账、下班倒计时、工作日进度、退休和存款，避免把一个仪表盘切成多个重复场景。
   - 主口号为“摸鱼有数，下班有期”，旁白用“日盼下班，终盼退休”收住长期视角。
3. **匿名看板（12.4–15.5 秒）**
   - 展示本地区间预览和匿名公开聚合结果，不展示个人原始记录。
4. **品牌收尾（15.5–20 秒）**
   - 展示主口号、支持平台和项目地址，不放难以阅读的长安装命令。

宣传片不再重复 README GIF 的隐私切换，避免“主动分享”和“强调隐藏”在同一条叙事里互相削弱。主口号固定为“摸鱼有数，下班有期”，“日盼下班，终盼退休”只用于退休场景。

### 竖版：浏览器前端

1. 以 Web 移动端响应式首屏“我的余薪”开场。
2. 用连续的浏览器真实快照展示今日入账变化，再纵向推进到公开样本。
3. 在“想看看大家”的旁白中依次展示聚合指标和匿名回声。
4. 结尾在浏览器画面上展示网页版与 GitHub 地址。

横版固定使用 TUI，竖版固定使用浏览器前端，发版更新素材时不得互换。竖版脚本默认启动本地 Web 并以生产数据模式截图；也可以通过 `YUXIN_SOCIAL_PAGE_URL` 指向已经部署的页面。浏览器截图只读取公开接口，不执行匿名贡献。

横版和竖版分别使用与画面对应的旁白文案与音轨，但保持同一类自然神经中文女声。音轨均随仓库保存，常规生成视频时无需联网；只有旁白变化、重新调用 `edge-tts` 时需要联网。设置 `YUXIN_PROMO_SILENT=1` 仍可生成无声横版。

### 横版宣传片旁白

语气自然、带一点打工人式自嘲，像同事间分享一个顺手的小工具，不使用播音腔。

| 时间 | 画面 | 旁白 |
| --- | --- | --- |
| 0–3.4 秒 | 情绪引入 | 这班不能白上，至少得知道赚了多少。 |
| 3.4–12.4 秒 | 完整仪表盘 | 今天赚了多少、还有多久下班、今年还要上多少天班，余薪都替你算好了。日盼下班，终盼退休。 |
| 12.4–15.5 秒 | 网页版 | 想看看大家，也只展示匿名聚合结果。 |

当前音轨保存在 `docs/assets/yuxin-promo-voice.m4a`。`render-promo.sh` 会自动混入；需要替换时，传入与分镜等长的音轨：

```bash
YUXIN_PROMO_VOICE_TRACK=/path/to/voice.m4a scripts/render-promo.sh
```

### 竖版宣传片分镜

| 时间 | 浏览器画面 | 旁白 |
| --- | --- | --- |
| 0–4 秒 | Web 首屏与“我的余薪” | 打开余薪网页版，先看看今天赚了多少。 |
| 4–11.8 秒 | 浏览器原生今日入账逐秒变化，镜头经过下班、盼头与躺平指标 | 工资会跟着时间往前跳。距离下班、下一个盼头，还有现在躺平能花多少，都在这一屏。 |
| 11.8–14 秒 | 公开样本宏观指标 | 再看看匿名样本…… |
| 14–18.1 秒 | 匿名回声模块 | ……听听打工人的匿名回声。这里只展示聚合结果。 |
| 18.1–22 秒 | 浏览器背景上的品牌与网址 | 余薪网页版。摸鱼有数，下班有期。 |

竖版使用独立的 `docs/assets/yuxin-social-voice.m4a`，由 `scripts/render-social-voice.sh` 通过新版 Edge Neural TTS 分段生成。默认使用 `zh-CN-XiaoxiaoNeural`、语速 `+12%`、音高 `-2Hz`，在每段末尾保留自然停顿。浏览器卡片中的今日入账由脚本连续截取网页自身的计算结果，不再使用 FFmpeg 重新绘制金额，避免字体、背景和数值口径与真实页面不一致。

## 自动化制作管线

```text
固定合成数据
    ↓
Go 演示会话按秒生成完整 ANSI 终端帧与临时 .cast
    ├── agg → README GIF
    └── agg/FFmpeg → 分镜裁切、缩放、字幕
                         + 本地配音音轨 → 带声宣传 MP4
```

- `scripts/demo-terminal-session.go` 负责确定性场景，每秒重算演示数据，不触碰本地配置。
- `scripts/render-demo.sh` 只负责生成 README 的 `docs/assets/yuxin-demo.gif`。
- 横版宣传视频使用 `scripts/render-promo.sh` 和 `docs/assets/yuxin-promo-voice.m4a`；竖版宣传视频使用 `scripts/render-social-cn.sh` 和 `docs/assets/yuxin-social-voice.m4a`。横版主体固定为 TUI，竖版主体固定为 Chrome/Chromium 渲染的 Web 前端；渲染需要带 `drawtext` 滤镜的 FFmpeg。
- `.cast`、中间帧和评审版 MP4 放在临时目录或 `dist/`，不进入正式 Release ZIP。
- 终端内容始终由 Go/agg 渲染；FFmpeg 只负责视频合成、裁切和转场。

## 验收标准

- 首帧是完整主页，GitHub 静态预览不出现空白画面。
- 工资、存款、年龄和退休信息全部来自固定合成数据。
- 中文、框线、进度条和双宽字符没有错位。
- 生成流程无需人工操作，可在同一环境重复得到一致场景。
- 演示工具不进入 `go.mod`，不增加余薪 CLI 和 Release ZIP 的体积。
- README GIF 加载轻快；完整 MP4 在手机和桌面端都能看清主要文字。
