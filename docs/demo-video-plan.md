# 余薪演示与宣传视频计划

## 已确定方向

- 长期使用 asciinema 录制固定终端会话，使用 agg 渲染 README GIF。
- 首帧直接展示完整主页，不使用空白终端或品牌片头作为封面。
- 所有素材只使用 `DemoDashboard` 固定合成数据，不读取用户配置，不访问网络。
- README GIF 和完整宣传视频是两个独立交付物，共用同一份演示数据和场景定义。
- Chrome Canvas 与 macOS 原生 Go 渲染器归档保留，只作为视觉和技术参考。

## 两类交付物

| 交付物 | 用途 | 建议规格 | 内容重点 |
| --- | --- | --- | --- |
| README GIF | GitHub 首页快速了解产品 | 约 10–15 秒、循环、无音频、控制在 1 MB 内 | 主页、隐私切换、分享卡 |
| 完整宣传视频 | Release、社交平台和项目介绍 | 1920×1080、约 20 秒、H.264 MP4、控制在 12 MB 内 | 打工人情绪、完整仪表盘、分享卡、品牌收尾 |

README 只嵌入 GIF。MP4 作为评审和宣传素材，不直接嵌入 README。

## README GIF 分镜

1. **主页（0–5 秒）**
   - 首帧即完整仪表盘。
   - 画面同时包含今日入账、下班倒计时和节假日进度，先让用户理解核心用途。
2. **隐私切换（5–8 秒）**
   - 切换到“隐藏金额和存款”，敏感数字变为占位符。
   - 底部状态同步显示隐私档位，证明截图前可以快速保护数据。
3. **分享卡（8–12 秒）**
   - 展示 `yuxin share` 的固定合成数据卡片。
   - 保留“无账号、离线运行、数据只在本地”。
4. **循环回主页**
   - 结尾不追加长说明，直接回到主页，避免循环时突兀。

## 完整宣传视频分镜

1. **情绪引入（0–3.4 秒）**
   - 以“这班不能白上，至少得知道赚了多少”建立打工人视角。
2. **完整仪表盘（3.4–12.4 秒）**
   - 在同一页面展示今日入账、下班倒计时、工作日进度、退休和存款，避免把一个仪表盘切成多个重复场景。
   - 主口号为“摸鱼有数，下班有期”，旁白用“日盼下班，终盼退休”收住长期视角。
3. **一键分享（12.4–15.5 秒）**
   - 分享卡快速出现，只说明一键生成和默认演示数据。
4. **品牌收尾（15.5–20 秒）**
   - 展示主口号、支持平台和项目地址，不放难以阅读的长安装命令。

宣传片不再重复 README GIF 的隐私切换，避免“主动分享”和“强调隐藏”在同一条叙事里互相削弱。主口号固定为“摸鱼有数，下班有期”，“日盼下班，终盼退休”只用于退休场景。

配音优先使用自然人声，背景音乐只作为低音量衬底；无声版本继续保留，方便 GitHub 和默认静音的社交平台使用。旁白稿见 [宣传片配音文案](promo-voiceover.md)。

## 自动化制作管线

```text
固定合成数据
    ↓
Go 演示会话输出 ANSI 终端场景
    ↓
asciinema 生成临时 .cast
    ├── agg → README GIF
    └── agg/FFmpeg → 分镜裁切、缩放、字幕 → 宣传 MP4
```

- `scripts/demo-terminal-session.go` 负责确定性场景，不触碰本地配置。
- `scripts/render-demo.sh` 只负责生成 README 的 `docs/assets/yuxin-demo.gif`。
- 完整宣传视频使用独立的 `scripts/render-promo.sh`，避免让轻量 GIF 流程承担视频排版逻辑。
- `.cast`、中间帧和评审版 MP4 放在临时目录或 `dist/`，不进入正式 Release ZIP。
- 终端内容始终由 asciinema/agg 渲染；FFmpeg 只负责视频合成、裁切和转场。

## 验收标准

- 首帧是完整主页，GitHub 静态预览不出现空白画面。
- 工资、存款、年龄和退休信息全部来自固定合成数据。
- 中文、框线、进度条和双宽字符没有错位。
- 生成流程无需人工操作，可在同一环境重复得到一致场景。
- 演示工具不进入 `go.mod`，不增加余薪 CLI 和 Release ZIP 的体积。
- README GIF 加载轻快；完整 MP4 在手机和桌面端都能看清主要文字。
