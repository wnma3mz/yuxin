# 余薪 Yuxin：终端 UI 方案

## 产品定位

一个本地优先、启动即看的实时工作仪表盘。默认运行 `yuxin` 进入持续刷新的 TUI；工资、工时、资产和身份信息只保存在本机。

默认首屏只解决两个问题：

1. 今天已经赚了多少，还要多久下班。
2. 下一个节假日还有多久。

退休和资产估算是可选模块。本年度节假日数据随发行包提供并载入内存，不随每秒刷新发起网络请求。

颜色只承担状态含义：绿色表示正在工作或数据正常，黄色表示午休/即将下班，灰色表示休息日，红色只用于配置或数据错误。无颜色终端仍能完整阅读。

## 可选模块

首次启动不要求用户提供出生日期或资产，退休和资产默认关闭。用户可以从按模块编辑的配置页随时开启。

程序可以手动录入身份证号码或出生日期，但不保存完整证件号码，只保存解析后的出生日期、性别和女性人员类型。

## 首次启动

首次启动不收集任何信息，自动写入默认 TOML 并直接进入主屏。默认值为月薪 8000 元、每月 22 个工作日、周一至周五、09:00–18:00、扣除 12:00–13:00 午休和每秒刷新。

## 退休计算边界

- 身份证可解析出生日期和性别，但不能判断女性属于原法定退休年龄 50 岁还是 55 岁的人群。女性用户必须额外选择人员类型；未选择前只显示“待确认人员类型”，不猜测退休时间。
- 按 2025 年起实施的渐进式延迟退休规则计算，并把规则表作为带版本的本地数据文件随程序发布，不在每次启动时请求外部 API。
- 主屏显示“预计退休年月”。“剩余天数”使用一个明确且固定的口径，例如距预计退休月第一天的日历天数，并在详情页说明，避免伪装成官方精确办理日期。
- 退休进度按 18 岁起算到预计退休月，不额外收集参加工作年份。
- 弹性提前或延迟退休不默认套用；可在详情配置中选择后作为单独情景展示。

## 资产与衍生指标

首版资产只录入名称、类型和余额：

- 银行卡活期
- 定期存款
- 其他现金类资产

核心口径：

```text
实时余额 = 所有启用账户余额之和 + 今日已经赚到的工资
可支配余额 = 实时余额 - 手动设置的应急保留金
现在退休每天可花 = 可支配余额 / 距退休日历天数
```

这些都是静态情景估算，不默认加入利息、通胀、涨薪、奖金、个税、养老金或未来支出。主屏只放四个可解释的数字，复杂假设以后再扩展。

## 交互与命令

默认命令：

```text
yuxin                 启动自动刷新仪表盘，默认每 1 秒刷新
yuxin --interval 2    临时改为每 2 秒刷新
yuxin once            输出一次快照后退出，方便脚本和 shell prompt 使用
yuxin config          进入交互式配置
yuxin doctor          检查配置、终端能力和可选数据源
yuxin update          校验并安装 GitHub 最新正式版
```

主屏快捷键：

```text
e  编辑配置     r  立即刷新     s  切换隐私演示
d  查看计算口径 ?  快捷键帮助   q  安全退出
```

刷新间隔默认是 `1.0` 秒，可在配置文件中长期修改，也可用 `--interval` 对本次运行临时覆盖。每次刷新重新计算实时收入，但不使用额外的高频动画。检测到非交互终端或输出被管道接收时，自动退化为 `once` 模式，避免 ANSI 控制符污染日志。

## 响应式终端

- `>= 100` 列：退休和资产左右双栏。
- `70–99` 列：退休、资产上下排列，保留完整指标。
- `< 70` 列：紧凑单列，只显示今日收入、下班倒计时、退休天数和每日可花；其余进入详情。
- 终端高度不足时优先隐藏说明文字，再隐藏次要衍生指标，不隐藏今日收入和退出提示。

## 实现边界

首版使用 Go 标准库实现，不包含第三方模块或运行时依赖。TUI 使用 ANSI 刷新，配置使用一个本地 TOML 文件。业务计算与终端渲染分离，使 `once` 和 TUI 共用同一份快照模型。

首版不做银行接口自动同步、不破解企业 SSO、不保存完整身份证号，也不做投资收益或养老金预测。
