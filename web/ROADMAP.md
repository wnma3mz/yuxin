# Yuxin 匿名数据看板路线

## 目标

将公开匿名数据看板与 CLI 匿名贡献能力推进到可上线、可发布状态。公开看板已上线；CLI 正式版本仍需单独授权发布。

## P0：产品口径

- [x] 采集字段、必填项、降精度口径与人民币币种。
- [x] 只公开聚合数据，不公开匿名原始行。
- [x] 不建立账号或个人记录；当前浏览器只凭随机凭证更新自己的原数值样本。
- [x] 匿名回声采用「建议 / 吐槽 / 许愿 / 打气」、80 字和先审核后公开。
- [x] 使用 GitHub Pages 默认地址，不配置自定义域名。
- [x] 永久不做账号系统、个人空间、排行和社交关系。
- [x] 允许当前浏览器保存本地配置，但不建立服务端个人档案、稳定身份或跨设备同步。

## P1：数据库与本地页面

- [x] 初始化 Vite + TypeScript 子项目和本地环境示例。
- [x] 完成公开聚合看板、本地区间比较、匿名提交和匿名回声。
- [x] 首屏增加动态演示余薪卡，保存本地配置后原地切换为实时个人数据。
- [x] 匿名贡献复用本地配置，存款与退休字段保持显式选择和默认关闭；浏览器不承担纯文本分享。
- [x] CLI 增加 `yuxin share`，最终确认前不联网；旧的 `--anonymous` 参数继续兼容。
- [x] 添加 Supabase schema、约束、最小权限和四个受限 RPC。
- [x] 原始数值和匿名回声使用独立请求、独立事务、分表保存且无应用层关联标识。
- [x] 客户端和数据库双重降精度，不保存应用层身份字段或精确时间。
- [x] 当天数据延迟公开；只按完整 10 份批次冻结发布，公开计数按 5 份分组，中位数按固定粒度取整。
- [x] 增加确认弹窗、隐私边界、错误和空状态。
- [x] 增加 Web 单元测试、SQL 权限测试、生产构建、依赖审计和 CSP。
- [x] 在目标 Supabase 项目执行全部 migration，并验证聚合 RPC、匿名回声类型和原始表权限。

## P2：GitHub Pages 与发布

- [x] 添加独立 Web CI 和 Pages 工作流，不改动 Go Release 流程。
- [x] 生产构建强制关闭 Mock，使用仓库 Actions Variable 注入 publishable key。
- [x] 增加冷启动演示状态：首个 10 份真实隐私批次公开前，展示固定演示数据并持续标记“非真实用户样本”。
- [x] 真实公开样本达到 10 份后原子切换为真实聚合数据；演示数据不写入 Supabase、不与真实数据混算。
- [x] 本地 Mock 页面经产品验收后，提交并推送代码。
- [x] 在 GitHub Pages 设置中选择 GitHub Actions，并配置 `SUPABASE_PUBLISHABLE_KEY`。
- [x] 验证 `/yuxin/` 子路径、移动端、键盘操作和无障碍标签。
- [x] 验证公开访客无法读取原始记录，只能调用受限 RPC。
- [x] 在隔离 PostgreSQL 中用测试数据验收提交、隔日聚合、分桶门槛和匿名回声审核并回滚；目标项目使用无写入探针验证 RPC 与权限。
- [x] 经单独授权后部署公开页面。
- [x] 更新版本号和正式 CHANGELOG，经单独授权后发布包含 CLI 匿名贡献能力的新版本。

## P3：公开运行后的治理

- [x] 增加匿名贡献纠错机制：由当前浏览器保存不可找回的匿名编辑凭证，只允许更新原提交，不引入账号或个人空间。
- [ ] 观察异常提交和统计漂移；若真实运行需要防滥用，在不建立账号、设备指纹或稳定身份的前提下评估限流或挑战层。
- [ ] 审核 Supabase API 日志内容和保留周期，避免主动扩充请求元数据。
- [ ] 制定原始贡献数据的保留和清理周期。
- [ ] 根据实际审核量决定是否需要独立的受限管理员控制台。
- [ ] 观察数据量后再决定是否增加更多维度或交叉分析。

## 已完成的上线操作

1. 验收本地 Mock 页面并授权提交、推送。
2. 在 Supabase 执行 migrations，并用无写入探针验证线上 RPC 与权限。
3. 将 publishable key 保存为仓库 Actions Variable `SUPABASE_PUBLISHABLE_KEY`。
4. 在仓库 Pages 设置中选择 GitHub Actions。
5. 完成公开 URL 的桌面端、移动端、交互、CSP 与水平溢出验收。

CLI v0.7.0 已发布。v0.8.0 已完成匿名纠错 migration、全量本地验证和宣传素材更新，正在执行最终发布流程。

## 已确认数据口径

1. 数据库只保存归一化后的粗粒度金额、工时、存款和整数退休年数，公开端只返回聚合结果。
2. 月薪、净工时、每周工作天数必填，其余字段选填；月薪沿用用户自行填写的口径。
3. 原始记录永不向公开角色开放。
4. 网页版可使用当前浏览器的随机凭证更新原数值样本；不提供记录列表、账号或凭证找回，匿名回声仍独立提交和审核。
5. 匿名回声最多 80 字，审核后公开且不关联数值。
6. 第一版只支持人民币和中文。
7. MVP 只做字段校验与隐私隔离，不承诺阻止同一来源重复或虚假提交；公开数据应被理解为匿名样本，不是统计抽样调查。
